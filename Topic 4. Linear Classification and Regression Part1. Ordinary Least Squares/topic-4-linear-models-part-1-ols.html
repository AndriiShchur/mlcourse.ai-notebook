Maximum Likelihood Estimation

One could ask why we choose to minimize the mean square error instead of something else? After all, one can minimize the mean absolute value of the residual. The only thing that will happen, if we change the minimized value, is that we will exceed the Gauss-Markov theorem conditions, and our estimates will therefore cease to be the optimal over the linear and unbiased ones.

Bias-Variance Decomposition

Let's talk a little about the error properties of linear regression prediction (in fact, this discussion is valid for all machine learning algorithms). We just covered the following:

  • true value of the target variable is the sum of a deterministic function f(x)f(x) and random error : y=f(x)+y=f(x)+;
  • error is normally distributed with zero mean and some variance: N(0,2)N(0,2);
  • true value of the target variable is also normally distributed: yN(f(x),2)yN(f(x),2);

Generally, as the model becomes more computational (e.g. when the number of free parameters grows), the variance (dispersion) of the estimate also increases, but bias decreases. Due to the fact that the training set is memorized completely instead of generalising, small changes lead to unexpected results (overfitting). On the other side, if the model is too weak, it will not be able to learn the pattern, resulting in learning something different that is offset with respect to the right solution.

Regularization of Linear Regression

There are situations where we might intentionally increase the bias of the model for the sake of stability i.e. to reduce the variance of the model Var(f)Var(f^). One of the conditions of the Gauss-Markov theorem is the full column rank of matrix XX. Otherwise, the OLS solution w=(XTX)1XTyw=(XTX)1XTy does not exist since the inverse matrix (XTX)1(XTX)1 does not exist. In other words, matrix XTXXTX will be singular or degenerate. This problem is called an ill-posed problem. Problems like this must be corrected, namely, matrix XTXXTX needs to become non-degenerate, or regular (which is why this process is called regularization). Often we observe the so-called multicollinearity in the data: when two or more features are strongly correlated, it is manifested in the matrix XX in the form of "almost" linear dependence between the columns. For example, in the problem of predicting house prices by their parameters, attributes "area with balcony" and "area without balcony" will have an "almost" linear relationship. Formally, matrix XTXXTX for such data is reversible, but, due to multicollinearity, some of its eigenvalues will be close to zero. In the inverse matrix XTXXTX, some extremely large eigenvalues will appear, as eigenvalues of the inverse matrix are 1i1i. The result of this vacillation of eigenvalues is an unstable estimate of model parameters, ie adding a new set of observations to the training data will lead to a completely different solution. One method of regularization is Tikhonov regularization, which generally looks like the addition of a new member to the mean squared error:

L(X,y,w)=12nyXw22+w2L(X,y,w)=12nyXw22+w2

The Tikhonov matrix is often expressed as the product of a number by the identity matrix: =2E=2E. In this case, the problem of minimizing the mean squared error becomes a problem with a restriction on the L2L2 norm. If we differentiate the new cost function with respect to the model parameters, set the resulting function to zero, and rearrange for ww, we get the exact solution of the problem.

w=(XTX+E)1XTyw=(XTX+E)1XTy

This type of regression is called ridge regression. The ridge is the diagonal matrix that we add to the XTXXTX matrix to ensure that we get a regular matrix as a result.

Such a solution reduces dispersion but becomes biased because the norm of the vector of parameters is also minimized, which makes the solution shift towards zero. It can be seen that by increasing the regularization parameter , we shift the solution towards zero.
